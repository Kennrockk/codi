Jerarquía en Css. -Resctricciones

Los navegadores aplican estilos a los documentos HTML por jerarquía basada en:

  1. La cascada
  2. La especificidad
  3. La herencia

La cascada es el procedimiento con el que un navegador aplica los estilos y el orden de importancia o restricción es el siguiente:

  1. Origen del código
    • Reglas del User Agent
    • Reglas del usuario
    • Reglas del desarrolador

  2. La especificidad del selector
  3. Orden de aparición

La especificidad es la fuerza restrictiva o peso de un selector.

  • Estilo en línea1
  • identificadores 0,1
  • Clases, atributos, pseudoclases 0,0,1
  • Etiquetas HTML 0,0,0,1
  • !important- Es el badass y pasa por encima de toda la jerarquía

La herencia es la capacidad que tienen los elementos padre de hacer heredara sus hijos sus propiedades. Valores inherit o heredados

Resetear navegadores

Ante la diferente interpretación de el aspecto de los elementos de HTML que utiliza cada navegador, nace la necesidad de borrar o resetear todas estas propiedades nativas de los navegadores para dejar el control a los desarrolladores.
Existen en el mercado diversos archivos Css que funcionan como reseteadores estandard, como normalize.css o reset.css.
Para más información: ver indexStyle.css

Posicionamiento

  • Fixed (fijo) Comportamiento nativo de posicionamiento de todos los elementos HTML.
  • Static (Estático)
  • Relative (Relativo)
  • Absolute (absoluto)
  • Sticky (adhesivo)
